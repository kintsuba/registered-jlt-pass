# 登録日本語教員パス MVP仕様

作成日: 2026-05-23

## 目的

「登録日本語教員パス」として、登録日本語教員試験の合格を目的にした資格勉強向けの問題演習サイトを作る。

教材を体系的に読むサイトではなく、公式出題範囲を軸に、予想問題を解き、解説を確認し、弱点を潰していくことを主目的にする。

## MVPの範囲

MVPでは選択式問題の演習に絞る。

対象に含めるもの:

- 予想問題ベースの選択式演習
- 公式出題範囲ベースのジャンル別演習
- ランダム演習
- 間違えた問題演習
- ブックマーク演習
- 解説表示
- ローカル進捗保存
- ログイン後のクラウド同期
- 学習記録

MVPでは対象外にするもの:

- 記述式対策
- 音声・聴解系の演習
- 模試形式
- ユーザー投稿問題
- 専用の問題管理画面
- 過去問演習の実提供

## デザイン・UI方針

想定ユーザーの9割以上がスマートフォンからのアクセスとなるため、**モバイルファースト**を大前提としてUIを設計する。

- PC表示時もモバイル版のレイアウトを中央に配置する、または間延びしない構成とする。
- 「押しやすさ（タップ領域の確保）」「片手での操作性」「スクロールのしやすさ」を最優先する。

## トップ画面の導線

トップ画面には以下の6導線を置く。

- ランダム問題
- 間違えた問題
- ジャンル別
- ブックマーク
- 学習記録
- 過去問

過去問はMVPでは準備中として表示し、クリック不可にする。

### トップ画面の動的メッセージ

トップ画面上部の見出しには、固定の挨拶（「こんにちは」など）ではなく、ユーザーの学習状況に応じたパーソナライズされた動的メッセージを表示する。本格的な実装は進捗データ管理機能の完成後に行う。

表示パターンの例：

- **継続学習**：「3日連続学習中！ / 素晴らしいペースです」
- **ネクストアクション提案**：「未正解が18問あります / 苦手克服から始めますか？」
- **休憩の提案**：「本日の演習: 50問 / 少し休憩しませんか？」
- **デフォルト**：「新しい一日です / 今日もコツコツ進めましょう」

## ページ構成

MVPのページ構成は以下とする。

```text
/
トップ・学習モード選択

/categories
ジャンル一覧
選択したジャンルで即セッション開始

/session/[id]
演習中

/result/[id]
演習結果

/records
学習記録

/login
メールリンクログイン

/settings
同期・ログイン・データ管理
```

`/practice/*` や `/categories/[id]` のような演習開始ページは作らない。対象が決まったら即セッションを開始する。

## 演習仕様

MVPの演習セッションは基本10問とする。

ジャンル別などで対象問題数が少ない場合は、問題数に応じて出題数を変える。

```text
0問:
開始不可

1〜9問:
全問出題

10〜19問:
10問抽選
直前セッション除外は可能な範囲
足りなければ直前分も戻す

20問以上:
直前セッションの問題を除外して10問抽選
```

回答後は、1問ごとに正誤と解説を表示する。

```text
問題表示
選択肢を選ぶ
回答する
正誤と解説を表示
次の問題へ進む
```

模試のように最後にまとめて採点する形式はMVPでは扱わない。

## 出題ロジック

### ランダム問題

全公開問題からランダムに10問出題する。

ただし、直前の演習セッションで出題した問題は除外する。問題数が足りない場合のみ、直前セッションの問題も候補に戻す。

### 間違えた問題

対象は未正解問題とする。

```text
hasAnswered = true
hasCorrect = false
```

一度でも正解した問題は、間違えた問題モードの対象から外す。

### ジャンル別

公式出題範囲ベースのカテゴリからジャンルを選び、選択したジャンル内で演習する。

問題数ごとの出題ルールは演習仕様に従う。

### ブックマーク

`bookmarked = true` の問題を対象にする。

問題数が1〜9問なら全問、10問以上なら10問抽選とする。直前セッション除外はジャンル別と同様に可能な範囲で適用する。

## 結果画面

演習終了後の結果画面には以下を表示する。

- 正解数 / 出題数
- 今回間違えた問題一覧
- 解説を見直す導線
- 同じモードでもう一度演習

「今回間違えた問題だけ再挑戦」ボタンはMVPでは入れない。復習はトップの間違えた問題モードに集約する。

## ブックマーク

各問題にブックマークボタンを置く。

ブックマークは以下のどちらの状態でも切り替え可能にする。

- 問題表示中
- 解説表示後

保存先はIndexedDBとし、ログイン済みの場合は同期対象に含める。

## 学習状態

MVPでは問題の学習状態を以下の4分類で扱う。

- 未回答
- 正解済み
- 未正解
- ブックマーク

内部的には以下のような進捗情報を持つ。

```text
hasAnswered
hasCorrect
attemptCount
wrongCount
lastAnsweredAt
bookmarked
```

## 学習記録

学習記録ページでは、MVPでは以下を表示する。

- 総解答数
- 正解済み問題数
- 未正解問題数
- ブックマーク数
- 分野別進捗
- 最近の演習結果

演習セッション履歴は `studySessions` として保存し、学習記録ページで利用する。

## 問題管理

問題管理はAirtableを第一候補とする。

Airtableは問題作成・レビュー用の管理元として使い、学習サイト本体がAirtable APIを直接参照しない構成にする。

```text
Airtable:
問題マスタ、カテゴリ、タグ、レビュー状態、公開管理

生成処理:
published の問題だけ取得し、アプリ用データへ正規化

Nuxt:
生成済みデータを配信・参照
```

専用の問題管理画面はMVPでは作らない。

## 問題データ

問題データには最低限以下を持たせる。

```text
id
status
officialCategory
tags
questionText
choices
correctChoice
explanation
choiceExplanations(optional)
sourceNote(optional)
referenceUrl(optional)
authorNote(optional)
```

全体解説は必須とする。選択肢ごとの個別解説は任意とし、存在する場合のみ表示する。

難易度フィールドはMVPでは持たせない。

### ステータス

問題ステータスは以下の4種類とする。

```text
draft
review
published
archived
```

アプリ配信対象は `published` のみとする。

`review` は現時点では必須運用しないが、将来のユーザー投稿や複数人作問に備えて用意しておく。

### 根拠メモ

以下の管理用フィールドを任意で用意する。

```text
sourceNote
referenceUrl
authorNote
```

ユーザー画面には基本的に表示せず、作問・レビュー用に使う。

## 問題更新と配信

問題データは1日1回、Airtableから取り込んでSSG生成する。

通常運用:

```text
Airtableで問題編集
1日1回 published 問題を取得
アプリ用データへ正規化
Nuxt SSG
Firebase Hostingへ配信
```

誤字修正や問題ミスに対応できるよう、必要時の手動再生成も用意する。

## 技術スタック

### フロントエンド

Nuxtを採用する。

ビルド・基盤はViteおよびvite-plusを前提にする。

### 配信

基本はSSGとする。

ログイン・クラウド同期の都合で必要になった場合のみ、後からサーバー/API構成を再検討する。

### ホスティング

Firebase Hostingを第一候補とする。

### 認証・DB

認証とDBは以下を第一候補とする。

```text
Firebase Auth
Firebase SQL/Data Connect
```

既存のFirebase有料プランを活かしつつ、NoSQLではなくPostgres系の設計を取れることを重視する。

試して不都合があれば、Supabase Auth + Postgres も比較候補として残す。

## 認証

MVPのログイン方法はメールリンクログインのみとする。

GoogleログインやSNSログインはMVPでは入れない。

想定ユーザーは大学院生〜社会人であり、資格学習サイトとしてはメールログインで十分とする。

## ローカル保存と同期

未ログインでもサイトの主機能を使えるようにする。

未ログイン時の学習データはIndexedDBに保存する。

ログイン後はクラウド同期を行い、複数端末で学習状態を復元できるようにする。

### 同期タイミング

1問ごとにはクラウド同期せず、IndexedDBへ即保存する。

クラウド同期は以下のタイミングで行う。

```text
ログイン直後:
クラウド取得 + ローカル統合

アプリ起動時:
ログイン済みならクラウドから取得

演習セッション完了時:
通常同期対象の差分をクラウドへ送信
```

途中終了時はクラウド同期しない。IndexedDB保存のみとし、次回以降の同期タイミングに回す。

### 初回ログイン時の扱い

未ログイン時のIndexedDBデータは、初回ログイン時に自動でクラウドへ統合する。

ユーザーに引き継ぎ確認を出して迷わせるのではなく、学習データを失わないことを優先する。

### 同期対象

通常同期対象:

```text
questionProgress
bookmarks
categoryProgress
studySettings
```

遅延同期対象:

```text
studySessions
```

`studySessions` は学習記録ページを開いたときだけ送受信する。

### 進捗マージ

`questionProgress` は問題ごとの進捗レコードとして同期する。

解答イベント全件ではなく、各問題の現在状態を保存する。

例:

```json
{
  "questionId": "q_001",
  "hasAnswered": true,
  "hasCorrect": true,
  "attemptCount": 2,
  "wrongCount": 1,
  "lastAnsweredAt": "2026-05-23T10:05:00Z"
}
```

マージルール:

```text
hasAnswered:
どちらかが true なら true

hasCorrect:
どちらかが true なら true

attemptCount:
加算

wrongCount:
加算

lastAnsweredAt:
新しいほう

bookmarked:
更新日時が新しいほう
```

## 演習セッション

演習セッション自体はローカルで管理する。

```text
sessionId
出題問題ID
回答状態
結果表示用データ
```

クラウドへ同期する通常データは、セッションそのものではなく、更新された `questionProgress` などとする。

ただし学習記録用の `studySessions` は保存する。

`studySessions` はセッションサマリのみを保存し、各問の選択肢や回答詳細までは保存しない。

例:

```json
{
  "sessionId": "sess_001",
  "mode": "random",
  "categoryId": null,
  "questionIds": ["q_001", "q_002"],
  "correctCount": 7,
  "questionCount": 10,
  "completedAt": "2026-05-23T10:00:00Z"
}
```

## ログイン誘導

未ログインでも演習できることを強みとするため、最初から強くログインを迫らない。

ログイン誘導は、未ログインで5回分の演習を完了したタイミングで表示する。

文言は、会員登録の強制ではなく、進捗のバックアップ・複数端末同期の案内にする。

## 設定ページ

設定ページでは以下を扱う。

ログイン済み:

- ログイン状態
- ログアウト
- 最終同期日時
- 手動同期
- アカウント削除

未ログイン:

- メールログイン案内
- この端末に学習データを保存している旨の表示

「端末内データだけ削除」はMVPでは表示しない。

アカウント削除時は以下をまとめて削除する。

- クラウド学習データ
- Firebase Authアカウント
- IndexedDB
